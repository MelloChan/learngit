2. 配置文件如何生效的
用一个我们大家都熟悉的例子来比较，就是css。一般来说，在一个页面中引入css文件的方法有三种，见：http://www.cnblogs.com/hutaoer/archive/2010/12/07/1899105.html，
通俗的讲就是行内样式“<div style=“width:200px”>”，内嵌样式“<style type=“text/css”>div{width:200px}</style>”
和外部样式“<link rel=“stylesheet” href=“div.css”>”。这三种样式的权重是行内>内嵌>外部。类似的，对于git来说，配置文件
的权重是仓库>全局>系统。Git会使用这一系列的配置文件来存储你定义的偏好，它首先会查找/etc/gitconfig文件（系统级），该
文件含有对系统上所有用户及他们所拥有的仓库都生效的配置值。接下来Git会查找每个用户的~/.gitconfig文件（全局级）。最后
Git会查找由用户定义的各个库中Git目录下的配置文件.git/config（仓库级），该文件中的值只对当前所属仓库有效。以上阐述的三
层配置从一般到特殊层层推进，如果定义的值有冲突，以后面层中定义的为准，例如：.git/config和/etc/gitconfig的较量中，
.git/config取得了胜利。虽然你也可以直接手动编辑这些配置文件，但是运行git config命令将会来得简单些。下面我们就来看看如
何利用git config命令来编辑配置各个级别的文件。
   3. 用git config命令查看配置文件
命令参数 Clist, 简写 -l
格式：git config [Clocal|Cglobal|Csystem] -l
查看仓库级的config，即.git/.config，命令：git config Clocal -l
查看全局级的config，即C:\Users\zuoyu.ht\.gitconfig，命令：git config Cglobal -l
查看系统级的config，即D:\Program Files\Git\etc\gitconfig，命令：git config Csystem -l
查看当前生效的配置，命令：git config -l，这个时候会显示最终三个配置文件计算后的配置信息，如下图：
   4. 使用git config命令编辑配置文件
编辑的英文单词是什么，没错，edit
命令参数 Cedit, 简写 -e
格式：git config [Clocal|Cglobal|Csystem] -e
查看仓库级的config，即.git/.config，命令：git config Clocal -e，与Clist参数不同的是，git config -e默认是编辑仓库级的配置文件。
查看全局级的config，即C:\Users\zuoyu.ht\.gitconfig，命令：git config Cglobal -e
查看系统级的config，即D:\Program Files\Git\etc\gitconfig，命令：git config Csystem -e
   执行这个命令的时候，git会用配置文件中设定的编辑器打开配置文件。
     5. 增加一个配置项
参数 Cadd
格式: git config [Clocal|Cglobal|Csystem] Cadd section.key value(默认是添加在local配置中)
注意add后面的section,key,value一项都不能少，否则添加失败。比如我们执行：git config Cadd cat.name tom

注意增加一项配置而不赋值 git config Cadd cat.age，或者单单增加一个section,git config Cadd cat1，不会成功。
6.获取一个配置项
有时候，我们并不需要查看所有配置的值，而是查看某个配置项的值，怎么做呢？
命令参数 Cget
格式：git config [Clocal|Cglobal|Csystem] Cget section.key(默认是获取local配置中内容)
我们先往global配置中写入一个cat.name=Tomcat的配置项，再使用git config Cget cat.name看看得到的是什么
结果就是local中的cat.name=Tom，因此git config Cget section.key 等价于git config Clocal Cget section.key
如果获取一个section不存在的key值，不会返回任何值
如果获取一个不存在的section的key值，则会报错
 7.删除一个配置项

格式：git config [Clocal|Cglobal|Csystem] Cunset section.key
